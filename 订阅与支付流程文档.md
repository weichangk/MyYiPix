# YiPix 订阅与支付流程文档

## 1. 概述

YiPix 平台的订阅与支付功能由两个独立微服务协作完成：

| 服务 | 职责 |
|---|---|
| **PaymentService** | 管理支付订单生命周期，对接 PayPal 支付（Orders API + Subscriptions API），处理 Webhook 回调 |
| **SubscriptionService** | 管理用户订阅生命周期（创建、激活、续期、取消、状态查询），消费支付完成事件自动激活订阅 |

两者通过 **RabbitMQ 事件总线** 进行异步解耦通信：
- **SubscriptionService** 作为 `PaymentCompletedEvent` 的消费者，支付完成后自动创建或续期订阅。
- **WebhookWorker** 同时消费 `PaymentCompletedEvent`，负责辅助任务（邮件通知、统计记录等）。

### 1.1 PayPal 公共模块

PayPal 集成逻辑封装在 `BuildingBlocks/PayPal` 公共模块中，供 PaymentService 和 SubscriptionService 共享：

```
BuildingBlocks/
└── PayPal/
    ├── IPayPalClient.cs           # 接口定义（6 个方法）
    ├── PayPalClient.cs            # HttpClient 封装（含 OAuth2 Token 缓存）
    ├── PayPalOptions.cs           # 配置类（含 PlanIdMappings、PlanPrices）
    ├── PayPalServiceExtensions.cs # DI 注册扩展方法
    └── Models/
        └── PayPalModels.cs        # 所有 PayPal API 请求/响应模型
```

## 2. 领域模型

### 2.1 PaymentService 实体

#### Payment（支付订单）

| 字段 | 类型 | 说明 |
|---|---|---|
| Id | Guid | 主键 |
| UserId | Guid | 关联用户 |
| Status | string | 状态：`Pending` → `Completed` / `Failed` / `Refunded` |
| Amount | decimal | 支付金额 |
| Currency | string | 货币（默认 USD） |
| PayPalOrderId | string? | PayPal 订单 ID |
| PayPalSubscriptionId | string? | PayPal 订阅 ID |
| PaymentType | string | `Subscription`（订阅付费）/ `OneTime`（一次性付费） |
| PlanId | string? | 关联的订阅计划 ID |
| CompletedAt | DateTime? | 完成时间 |
| FailureReason | string? | 失败原因 |

#### WebhookLog（Webhook 回调日志）

| 字段 | 类型 | 说明 |
|---|---|---|
| EventType | string | 事件类型（如 `PAYMENT.CAPTURE.COMPLETED`） |
| ResourceId | string | 资源 ID（用于幂等去重） |
| Payload | string | 原始 JSON 载荷 |
| Processed | bool | 是否已处理 |
| ProcessingError | string? | 处理错误信息 |
| ProcessedAt | DateTime? | 处理完成时间 |

### 2.2 SubscriptionService 实体

#### Subscription（订阅）

| 字段 | 类型 | 说明 |
|---|---|---|
| Id | Guid | 主键 |
| UserId | Guid | 关联用户 |
| Plan | string | 计划：`Free` / `Monthly` / `Yearly` / `Lifetime` |
| Status | string | 状态：`Active` / `Cancelled` / `Expired` / `PastDue` / `Suspended` |
| StartDate | DateTime | 订阅开始时间 |
| EndDate | DateTime? | 到期时间（Lifetime 为 null） |
| PayPalSubscriptionId | string? | PayPal 订阅 ID |
| CancelledAt | DateTime? | 取消时间 |
| CancellationReason | string? | 取消原因 |
| IsActive | bool (计算) | `Status == "Active" && (EndDate == null \|\| EndDate > Now)` |

**到期时间计算规则：**
- Monthly → 当前时间 + 1 个月
- Yearly → 当前时间 + 1 年
- Lifetime → 无到期时间（null）
- Free → 无到期时间（null）

#### SubscriptionHistory（订阅变更历史）

| 字段 | 类型 | 说明 |
|---|---|---|
| SubscriptionId | Guid | 关联的订阅 |
| UserId | Guid | 关联用户 |
| FromStatus | string | 变更前状态 |
| ToStatus | string | 变更后状态 |
| Reason | string? | 变更原因 |

## 3. 集成事件

定义在 `BuildingBlocks/Contracts/Events/IntegrationEvents.cs`：

| 事件 | 参数 | 发布方 | 消费方 |
|---|---|---|---|
| `PaymentCompletedEvent` | UserId, PaymentId, Amount, Currency, **PlanId**, **PaymentType** | PaymentService | SubscriptionService（激活/续期订阅）、WebhookWorker（邮件/统计） |
| `PaymentFailedEvent` | UserId, PaymentId, Reason | PaymentService（捕获失败时发布） | — |
| `SubscriptionActivatedEvent` | UserId, SubscriptionId, PlanType | SubscriptionService | 无消费者（可扩展用于通知等） |
| `SubscriptionCancelledEvent` | UserId, SubscriptionId | SubscriptionService | 无消费者（可扩展用于通知等） |
| `SubscriptionExpiredEvent` | UserId, SubscriptionId | （定义但未使用，待后台定时任务实现） | — |

> **变更说明**：`PaymentCompletedEvent` 已扩展为 6 个参数，新增 `PlanId`（订阅计划标识，如 `Monthly`/`Yearly`/`Lifetime`）和 `PaymentType`（支付类型：`Subscription` 或 `OneTime`），用于下游 SubscriptionService 根据支付信息自动激活或续期订阅。

## 4. 核心业务流程

### 4.1 新用户订阅购买流程

```
┌──────────┐    ┌────────────────┐    ┌─────────────────────┐    ┌───────────────┐
│  前端客户端  │    │ PaymentService │    │   SubscriptionSvc   │    │ WebhookWorker │
└─────┬────┘    └───────┬────────┘    └──────────┬──────────┘    └──────┬────────┘
      │                  │                        │                      │
      │ ① POST /api/payments                     │                      │
      │  {UserId, PlanId, ReturnUrl, CancelUrl}   │                      │
      │─────────────────>│                        │                      │
      │                  │                        │                      │
      │                  │ 创建 Payment (Pending)  │                      │
      │                  │ 根据 PlanId 判断：       │                      │
      │                  │  Monthly/Yearly →        │                      │
      │                  │    PayPal Subscriptions API                    │
      │                  │  Lifetime →              │                      │
      │                  │    PayPal Orders API     │                      │
      │                  │ 获取 ApproveUrl          │                      │
      │   返回 PaymentDto│                        │                      │
      │   (含 ApproveUrl)│                        │                      │
      │<─────────────────│                        │                      │
      │                  │                        │                      │
      │ ② 重定向到 PayPal 页面完成支付授权          │                      │
      │                  │                        │                      │
      │ ③ (仅一次性付费) POST /api/payments/capture │                      │
      │  {PayPalOrderId}  │                        │                      │
      │─────────────────>│                        │                      │
      │                  │                        │                      │
      │                  │ 调用 PayPal             │                      │
      │                  │ Capture API             │                      │
      │                  │ 更新 Status→Completed   │                      │
      │                  │                        │                      │
      │                  │──PaymentCompletedEvent─>│                      │
      │                  │   (via RabbitMQ)        │──────────────────────>│
      │   返回 PaymentDto│                        │                      │
      │<─────────────────│                        │                      │
      │                  │                        │                      │
      │                  │                        │ ④ ActivateOrRenew-    │
      │                  │                        │   ByPaymentAsync      │
      │                  │                        │ 根据 PlanId 创建或     │
      │                  │                        │ 续期订阅               │
      │                  │                        │                      │
      │                  │                        │ 发布                   │
      │                  │                        │ SubscriptionActivated │
      │                  │                        │ Event                 │
      │                  │                        │                      │
      │                  │                        │                      │ ⑤ 辅助任务
      │                  │                        │                      │ TODO: 发送确认邮件
      │                  │                        │                      │ TODO: 记录统计数据
      │                  │                        │                      │
```

> **说明**：
> - `PaymentCompletedEvent` 同时被 SubscriptionService 和 WebhookWorker 消费（RabbitMQ 不同队列）。
> - 订阅型支付（Monthly/Yearly）由 PayPal 自动激活并通过 Webhook 通知，前端无需调用 Capture。
> - 一次性付费（Lifetime）需前端回调后手动调用 Capture API。

### 4.2 PayPal Webhook 回调流程

```
┌──────────┐    ┌────────────────┐
│  PayPal   │    │ PaymentService │
└─────┬────┘    └───────┬────────┘
      │                  │
      │ POST /api/payments/webhook (AllowAnonymous)
      │  Headers: PAYPAL-TRANSMISSION-ID, PAYPAL-TRANSMISSION-TIME,
      │           PAYPAL-CERT-URL, PAYPAL-AUTH-ALGO, PAYPAL-TRANSMISSION-SIG
      │  Body: { event_type, resource, ... }
      │─────────────────>│
      │                  │
      │                  │ ① 读取原始 Body
      │                  │ ② 验证 PayPal Webhook 签名 ✅
      │                  │    (调用 PayPal Verify API)
      │                  │    WebhookId 为空时跳过验签(开发环境)
      │                  │ ③ 解析 JSON 提取 event_type 和 resource.id ✅
      │                  │ ④ 幂等性检查 (EventType + ResourceId)
      │                  │    → 如果已处理则跳过
      │                  │ ⑤ 根据事件类型分发处理 ✅：
      │                  │    - PAYMENT.CAPTURE.COMPLETED → 标记支付完成
      │                  │    - PAYMENT.CAPTURE.DENIED → 记录日志
      │                  │    - PAYMENT.CAPTURE.REFUNDED → 记录日志
      │                  │    - BILLING.SUBSCRIPTION.ACTIVATED → 激活订阅
      │                  │    - BILLING.SUBSCRIPTION.CANCELLED → 记录日志
      │                  │    - BILLING.SUBSCRIPTION.SUSPENDED → 记录日志
      │                  │    - PAYMENT.SALE.COMPLETED → 续费记录
      │                  │ ⑥ 记录 WebhookLog
      │   200 OK         │
      │<─────────────────│
```

### 4.3 取消订阅流程

```
┌──────────┐    ┌───────────────────┐    ┌────────┐
│  前端客户端  │    │SubscriptionService│    │ PayPal  │
└─────┬────┘    └────────┬──────────┘    └───┬────┘
      │                   │                   │
      │ POST /api/subscriptions/{id}/cancel?reason=xxx
      │──────────────────>│                   │
      │                   │                   │
      │                   │ ① 更新 Status → Cancelled
      │                   │ ② 记录 CancelledAt 和 CancellationReason
      │                   │ ③ 写入 SubscriptionHistory
      │                   │    (FromStatus → Cancelled, Reason)
      │                   │ ④ 调用 PayPal Cancel Subscription API ✅
      │                   │    (失败不阻塞本地取消)
      │                   │──────────────────>│
      │                   │                   │
      │                   │ ⑤ 发布 SubscriptionCancelledEvent
      │   200 OK          │                   │
      │<──────────────────│                   │
```

> **说明**：取消订阅时会同步调用 PayPal Cancel Subscription API 取消 PayPal 侧的自动扣款。如果 PayPal 调用失败（网络异常等），本地取消操作不受影响，仅记录警告日志。

## 5. API 端点一览

### 5.1 PaymentService（`/api/payments`）

| 方法 | 路由 | 认证 | 请求体 | 说明 |
|---|---|---|---|---|
| GET | `/{id}` | ✅ JWT | — | 查询单笔支付记录 |
| GET | `/user/{userId}` | ✅ JWT | — | 查询用户所有支付记录 |
| POST | `/` | ✅ JWT | `CreatePaymentRequest` | 创建支付订单 |
| POST | `/capture` | ✅ JWT | `CapturePaymentRequest` | 捕获（确认）PayPal 支付 |
| POST | `/webhook` | ❌ 匿名 | Raw Body (JSON) | PayPal Webhook 回调 |

**请求/响应 DTO：**

```csharp
// 创建支付
record CreatePaymentRequest(Guid UserId, string PlanId, string ReturnUrl, string CancelUrl);

// 捕获支付
record CapturePaymentRequest(string PayPalOrderId);

// 支付响应（新增 ApproveUrl 字段）
record PaymentDto(Guid Id, Guid UserId, decimal Amount, string Currency,
                  PaymentStatus Status, string PayPalOrderId, DateTime CreatedAt,
                  string? ApproveUrl = null);

// 支付状态枚举
enum PaymentStatus { Pending, Completed, Failed, Refunded }
```

> **说明**：`PaymentDto.ApproveUrl` 仅在创建支付时返回，用于前端重定向到 PayPal 授权页面。`PayPalOrderId` 字段复用为 `PayPalOrderId ?? PayPalSubscriptionId`。

### 5.2 SubscriptionService（`/api/subscriptions`）

| 方法 | 路由 | 认证 | 请求体 | 说明 |
|---|---|---|---|---|
| GET | `/user/{userId}` | ✅ JWT | — | 查询用户所有订阅记录 |
| GET | `/user/{userId}/active` | ✅ JWT | — | 查询用户当前活跃订阅 |
| GET | `/user/{userId}/status` | ✅ JWT | — | 检查用户订阅状态 |
| POST | `/` | ✅ JWT | `CreateSubscriptionRequest` | 创建新订阅 |
| POST | `/{id}/cancel` | ✅ JWT | Query: `reason` | 取消订阅 |

**请求/响应 DTO：**

```csharp
// 创建订阅（每个用户同时只能有一个活跃订阅）
record CreateSubscriptionRequest(Guid UserId, SubscriptionPlan Plan, string PayPalSubscriptionId);

// 订阅响应
record SubscriptionDto(Guid Id, Guid UserId, SubscriptionPlan Plan, SubscriptionStatus Status,
                       DateTime StartDate, DateTime? EndDate, string? PayPalSubscriptionId);

// 订阅状态查询响应
record SubscriptionStatusResponse(bool IsActive, SubscriptionPlan Plan, DateTime? ExpiresAt);

// 订阅计划枚举
enum SubscriptionPlan { Free, Monthly, Yearly, Lifetime }

// 订阅状态枚举
enum SubscriptionStatus { Active, Cancelled, Expired, PastDue, Suspended }
```

## 6. 业务规则

| 规则 | 说明 |
|---|---|
| **单活跃订阅** | 每个用户同时只能有一个 Active 状态的订阅，重复创建会返回 `409 Conflict` |
| **Webhook 幂等** | 通过 `(EventType, ResourceId)` 联合索引去重，已处理的事件会被跳过 |
| **支付状态流转** | `Pending` → `Completed`（捕获成功）/ `Failed`（捕获失败）/ `Refunded`（退款） |
| **订阅状态流转** | `Active` → `Cancelled`（用户取消）/ `Expired`（到期）/ `PastDue`（逾期）/ `Suspended`（暂停） |
| **状态变更审计** | 每次订阅状态变更都会写入 `SubscriptionHistory`，记录 From/To/Reason |
| **计划类型路由** | `Monthly`/`Yearly` → PayPal Subscriptions API（周期性扣款）；`Lifetime` → PayPal Orders API（一次性扣款） |
| **订阅续期逻辑** | 续费时若已有活跃订阅则延长 `EndDate`，否则创建新订阅。续期时间从当前 `EndDate` 或 `UtcNow` 中取较晚者 |
| **PayPal 联动取消** | 取消订阅时同步调用 PayPal Cancel API，失败不阻塞本地操作 |
| **Webhook 签名验证** | 生产环境必须配置 `WebhookId`；为空时跳过验签（仅限开发环境） |
| **Token 缓存** | PayPal OAuth2 Token 缓存在内存中，提前 60 秒刷新，使用 `SemaphoreSlim` 保证并发安全 |

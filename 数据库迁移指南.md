# 数据库迁移指南

## 概述

本项目每个微服务拥有独立的 `DbContext` 和 `Migrations` 目录，互不影响。数据库迁移使用 Entity Framework Core 的 `dotnet ef` 工具管理。

## 前置条件

安装 EF Core CLI 工具：

```powershell
dotnet tool install --global dotnet-ef
```

## 初始化迁移（首次建表）

为每个服务生成初始迁移文件，在项目根目录执行：

```powershell
dotnet ef migrations add InitialCreate --project src/Services/AuthService
dotnet ef migrations add InitialCreate --project src/Services/UserService
dotnet ef migrations add InitialCreate --project src/Services/SubscriptionService
dotnet ef migrations add InitialCreate --project src/Services/PaymentService
dotnet ef migrations add InitialCreate --project src/Services/DownloadService
dotnet ef migrations add InitialCreate --project src/Services/ProductService
dotnet ef migrations add InitialCreate --project src/Services/AnalyticsService
dotnet ef migrations add InitialCreate --project src/Services/TaskService
dotnet ef migrations add InitialCreate --project src/Services/FileService
```

> 已有 `Migrations/` 目录的服务无需重复执行。

## 后续更新数据库（新增字段、新增表等）

以 AuthService 为例，假设给 `User` 实体新增了 `PhoneNumber` 字段：

### 1. 生成新的迁移

```powershell
dotnet ef migrations add AddPhoneNumber --project src/Services/AuthService
```

迁移名称应描述此次变更，如 `AddPhoneNumber`、`AddOrderTable`、`RemoveDeprecatedFields` 等。

### 2. 应用迁移

#### 本地开发（dotnet run）

不需要手动执行。每个服务的 `Program.cs` 中已包含：

```csharp
if (app.Environment.IsDevelopment())
{
    using var scope = app.Services.CreateScope();
    var db = scope.ServiceProvider.GetRequiredService<XxxDbContext>();
    await db.Database.MigrateAsync();
}
```

服务启动时会自动应用所有未执行的迁移。

#### Docker 开发环境

重新构建即可，容器启动后同样会自动迁移：

```powershell
cd docker
docker compose up --build -d
```

#### 生产环境

`MigrateAsync()` 仅在 `ASPNETCORE_ENVIRONMENT=Development` 时执行，生产环境需要手动迁移：

```powershell
# 方式 A：本地连接生产数据库执行
dotnet ef database update --project src/Services/AuthService \
  --connection "Host=生产地址;Port=5432;Database=yipix;Username=xxx;Password=xxx"

# 方式 B：进入容器执行（需容器中有 ef 工具）
docker exec -it yipix-auth dotnet ef database update
```

> **重要**：生产环境更新前务必先备份数据库。

## 常用命令速查

| 操作 | 命令 |
|------|------|
| 生成迁移 | `dotnet ef migrations add <名称> --project src/Services/<服务名>` |
| 应用迁移 | `dotnet ef database update --project src/Services/<服务名>` |
| 回滚到指定迁移 | `dotnet ef database update <迁移名> --project src/Services/<服务名>` |
| 删除最近一次未应用的迁移 | `dotnet ef migrations remove --project src/Services/<服务名>` |
| 查看迁移列表 | `dotnet ef migrations list --project src/Services/<服务名>` |
| 生成 SQL 脚本（不直接执行） | `dotnet ef migrations script --project src/Services/<服务名>` |

## 注意事项

1. **迁移文件必须提交到代码仓库**，它们是数据库 schema 的版本记录
2. 每个微服务的迁移**独立管理**，修改 AuthService 的实体不会影响 UserService 的数据库
3. 不要手动修改已应用的迁移文件，如需修正应生成新的迁移
4. 生产环境建议使用 `dotnet ef migrations script` 生成 SQL 脚本审查后再执行
5. 多人协作时，如果迁移文件冲突，需要删除冲突的迁移后重新生成

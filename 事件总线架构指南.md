# YiPix 事件总线架构指南

## 一、概述

YiPix 微服务间通过 **RabbitMQ 事件总线** 实现松耦合的异步通信。服务通过发布集成事件通知其他服务，消费方通过订阅事件并注册 Handler 来响应。

核心代码位于 `src/BuildingBlocks/EventBus/`，事件定义位于 `src/BuildingBlocks/Contracts/Events/`。

## 二、核心组件

### 2.1 架构图

```
发布方                        RabbitMQ                          消费方
  │                              │                                │
  │  PublishAsync<T>(event)      │                                │
  │─────────────────────────────>│                                │
  │                              │  Exchange: yipix_event_bus     │
  │                              │  Type: Topic                   │
  │                              │  RoutingKey: 事件类名           │
  │                              │                                │
  │                              │  Queue: {事件名}_queue          │
  │                              │───────────────────────────────>│
  │                              │                                │
  │                              │              Handler.HandleAsync(@event)
  │                              │                  ↓ 成功 → ACK
  │                              │                  ↓ 失败 → NACK（重入队列）
```

### 2.2 关键文件

| 文件 | 说明 |
|------|------|
| `EventBus/Abstractions/IEventBus.cs` | 定义 `IEventBus` 接口（`PublishAsync` + `Subscribe`）和 `IIntegrationEventHandler<T>` |
| `EventBus/RabbitMQ/RabbitMQEventBus.cs` | RabbitMQ 实现，消息 JSON 序列化，持久化投递，手动 ACK |
| `EventBus/EventBusServiceExtensions.cs` | DI 扩展方法 `AddRabbitMQEventBus(connectionString)` |
| `Contracts/Events/IntegrationEvents.cs` | 所有集成事件的 record 定义 |

### 2.3 IEventBus 接口

```csharp
public interface IEventBus
{
    // 发布事件到 RabbitMQ
    Task PublishAsync<T>(T @event, CancellationToken ct = default) where T : IntegrationEvent;

    // 订阅事件，绑定 Handler
    void Subscribe<T, THandler>()
        where T : IntegrationEvent
        where THandler : IIntegrationEventHandler<T>;
}
```

## 三、集成事件清单

所有事件继承自 `IntegrationEvent`，自带 `Id`（Guid）和 `OccurredOn`（UTC 时间戳）。

### Auth 事件

| 事件 | 参数 | 触发时机 |
|------|------|---------|
| `UserCreatedEvent` | UserId, Email, DisplayName | 用户注册成功 |
| `UserLoggedInEvent` | UserId, IpAddress, DeviceType | 用户登录成功 |

### Subscription 事件

| 事件 | 参数 | 触发时机 |
|------|------|---------|
| `SubscriptionActivatedEvent` | UserId, SubscriptionId, PlanType | 创建/激活订阅 |
| `SubscriptionCancelledEvent` | UserId, SubscriptionId | 取消订阅 |
| `SubscriptionExpiredEvent` | UserId, SubscriptionId | 订阅过期 |

### Payment 事件

| 事件 | 参数 | 触发时机 |
|------|------|---------|
| `PaymentCompletedEvent` | UserId, PaymentId, Amount, Currency | 支付完成 |
| `PaymentFailedEvent` | UserId, PaymentId, Reason | 支付失败 |

### Download 事件

| 事件 | 参数 | 触发时机 |
|------|------|---------|
| `DownloadStartedEvent` | UserId, Version, Platform | 生成下载链接 |
| `DownloadCompletedEvent` | UserId, Version, Platform | 下载完成 |

### Task 事件

| 事件 | 参数 | 触发时机 |
|------|------|---------|
| `TaskCreatedEvent` | TaskId, UserId, TaskType | 创建图片处理任务 |
| `TaskCompletedEvent` | TaskId, UserId, TaskType | 任务处理完成 |
| `TaskFailedEvent` | TaskId, UserId, Reason | 任务处理失败 |

## 五、使用方式

### 5.1 发布事件（在微服务中）

```csharp
// 1. 注入 IEventBus
private readonly IEventBus _eventBus;

// 2. 发布事件
await _eventBus.PublishAsync(new UserCreatedEvent(user.Id, user.Email, user.DisplayName), ct);
```

### 5.2 注册事件总线（在 Program.cs 中）

```csharp
// 所有服务统一注册方式
builder.Services.AddRabbitMQEventBus(
    builder.Configuration.GetConnectionString("RabbitMQ") ?? "amqp://guest:guest@localhost:5672");
```

### 5.3 消费事件（在 Worker 中）

**步骤一：实现 Handler**

```csharp
public class TaskCreatedEventHandler : IIntegrationEventHandler<TaskCreatedEvent>
{
    public async Task HandleAsync(TaskCreatedEvent @event, CancellationToken ct = default)
    {
        // 处理逻辑
    }
}
```

**步骤二：注册 Handler 到 DI**

```csharp
// Worker 的 Program.cs
services.AddScoped<TaskCreatedEventHandler>();
```

**步骤三：订阅事件**

```csharp
// 在 BackgroundService 的 ExecuteAsync 中调用
var eventBus = _serviceProvider.GetRequiredService<IEventBus>();
eventBus.Subscribe<TaskCreatedEvent, TaskCreatedEventHandler>();
```

## 六、可靠性保障

| 机制 | 说明 |
|------|------|
| **消息持久化** | `DeliveryModes.Persistent`，RabbitMQ 重启后消息不丢失 |
| **持久化队列** | `durable: true`，队列定义持久化 |
| **手动 ACK** | 处理成功后才确认消息（`BasicAckAsync`） |
| **失败重入队列** | 处理失败时 NACK 并 requeue（`BasicNackAsync(tag, false, true)`） |
| **消息 ID** | 每条消息携带事件 `Id` 和时间戳，便于追踪和幂等处理 |

## 七、RabbitMQ 配置

| 配置 | 值 |
|------|-----|
| Exchange 名称 | `yipix_event_bus` |
| Exchange 类型 | `Topic` |
| 队列命名规则 | `{EventName}_queue`（如 `UserCreatedEvent_queue`） |
| RoutingKey | 事件类名（如 `UserCreatedEvent`） |
| 开发环境连接 | `amqp://guest:guest@localhost:5672` |
| 管理界面 | `http://localhost:15672`（guest/guest） |

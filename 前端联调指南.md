# 前端联调指南

本指南介绍如何让前端开发人员访问后端 API 接口进行联调开发。

---

## 方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **A. 局域网共享** | 同一 WiFi / 公司内网 | 零成本、速度快 | 你电脑关机就没法用 |
| **B. 内网穿透** | 不同网络（远程协作） | 不限网络 | 免费版有限制，地址可能变 |
| **C. 生产服务器** | 所有场景（推荐） | 最稳定，24h 在线 | 代码更新需重新部署 |

---

## 方案 A：局域网共享（同一网络）

适用于你和前端同事在同一个局域网内（公司网络 / 同一 WiFi）。

### 1. 查看本机 IP

```powershell
ipconfig
```

找到当前网络适配器的 IPv4 地址，例如 `192.168.1.100`。

### 2. 启动服务

```powershell
# 启动基础设施
cd docker
docker compose -f docker-compose.infra.yml up -d

# 回到项目根目录，一键启动所有服务
cd ..
.\start-all.ps1
```

或启动单个服务：

```powershell
dotnet run --project src/Services/AuthService
```

### 3. 开放 Windows 防火墙

以**管理员身份**运行 PowerShell：

```powershell
# 开放服务端口范围（一次性操作）
New-NetFirewallRule -DisplayName "YiPix API Ports" -Direction Inbound -LocalPort 5000-5400 -Protocol TCP -Action Allow
```

> 不再需要时可删除规则：
> ```powershell
> Remove-NetFirewallRule -DisplayName "YiPix API Ports"
> ```

### 4. 前端同事访问

将 `192.168.1.100` 替换为你的实际 IP：

```
http://192.168.1.100:5282/api/auth/...
http://192.168.1.100:{port}/api/users/...
```

前端项目中配置 API base URL：

```javascript
// .env.development
VITE_API_BASE_URL=http://192.168.1.100:5282
```

> ⚠️ 注意：你的电脑关机或服务停止后，前端将无法访问。

## 方案 B：内网穿透（不同网络）

适用于远程协作，前端同事不在同一局域网。

### 使用 cpolar（国内推荐）

1. 注册并安装 cpolar：https://www.cpolar.com

2. 启动隧道：
   ```powershell
   cpolar http 5282
   ```

3. 得到公网地址，例如 `https://xxxxxxxx.cpolar.top`

4. 前端同事使用该地址访问：
   ```
   https://xxxxxxxx.cpolar.top/api/auth/...
   ```

### 使用 ngrok（国外服务）

```powershell
ngrok http 5282
```

得到类似 `https://xxxx.ngrok-free.app` 的地址。

> ⚠️ 免费版限制：
> - 地址每次启动会变
> - 有连接数/带宽限制
> - 每个隧道只能转发一个端口（多服务需开多个隧道或用方式 C 的 Docker + Nginx）

## 方案 C：使用生产服务器（推荐）

你的服务器已部署好，前端同事直接使用生产域名即可，无需依赖你的电脑。

### API 地址

```
https://yipixapi.lilikk.com
```

### 前端项目配置

```javascript
// .env.development 或 .env.production
VITE_API_BASE_URL=https://yipixapi.lilikk.com
```

### 接口示例

```bash
# 注册
curl -X POST https://yipixapi.lilikk.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"123456"}'

# 登录
curl -X POST https://yipixapi.lilikk.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"123456"}'
```

## CORS 跨域配置

前端通过浏览器访问 API 时可能遇到跨域问题。确保后端服务已配置 CORS 允许前端域名。

如果前端运行在 `http://localhost:3000`，后端需要允许该来源。各服务的 `Program.cs` 中应包含类似配置：

```csharp
builder.Services.AddCors(options =>
{
    options.AddDefaultPolicy(policy =>
    {
        policy.WithOrigins(
            "http://localhost:3000",    // 前端本地开发
            "https://yipix.lilikk.com"  // 前端生产域名
        )
        .AllowAnyHeader()
        .AllowAnyMethod()
        .AllowCredentials();
    });
});
```

> 如果遇到跨域报错，优先检查后端 CORS 配置。

# ASP.NET Core 配置优先级说明

## 背景

在使用 `docker compose up --build -d` 启动服务时，存在多个配置来源。本文档说明它们的加载顺序和覆盖关系。

## 配置加载顺序（优先级从低到高）

```
appsettings.json          ← 最先加载，优先级最低
       ↓ 覆盖
appsettings.{Env}.json    ← 根据 ASPNETCORE_ENVIRONMENT 决定加载哪个
       ↓ 覆盖
环境变量                   ← 优先级最高，最终生效
```

ASP.NET Core 的配置系统是**叠加式（layered）** 的，后加载的配置会覆盖先加载的同名配置项，未被覆盖的配置项继续保留。

## 具体场景分析

### Docker Compose 开发环境

`docker-compose.yml` 中设置了：

```yaml
environment:
  - ASPNETCORE_ENVIRONMENT=Development
  - CdnSettings__BaseUrl=https://cdn.yipix.com
  - CdnSettings__AuthKey=your-cdn-auth-secret-key
  - CdnSettings__LinkExpirationMinutes=60
```

加载过程：

| 步骤 | 配置来源 | 说明 |
|------|---------|------|
| 1 | `appsettings.json` | 加载所有基础配置 |
| 2 | `appsettings.Development.json` | 因为 `ASPNETCORE_ENVIRONMENT=Development`，加载并覆盖同名项 |
| 3 | Docker 环境变量 | `CdnSettings__AuthKey` 等环境变量覆盖前两步的同名值 |

### Docker Compose 生产环境

`docker-compose.prod.yml` 中设置了：

```yaml
environment:
  - ASPNETCORE_ENVIRONMENT=${ASPNETCORE_ENVIRONMENT}   # 值来自 .env 文件，通常为 Production
  - CdnSettings__BaseUrl=${CDN_BASE_URL}
  - CdnSettings__AuthKey=${CDN_AUTH_KEY}
  - CdnSettings__LinkExpirationMinutes=${CDN_LINK_EXPIRATION_MINUTES:-60}
```

加载过程：

| 步骤 | 配置来源 | 说明 |
|------|---------|------|
| 1 | `appsettings.json` | 加载基础配置 |
| 2 | `appsettings.Production.json` | 如果存在则加载（当前项目未提供此文件） |
| 3 | Docker 环境变量（来自 `.env`） | 最终生效值，敏感信息通过 `.env` 注入 |

### 本地直接运行（不使用 Docker）

```bash
dotnet run --project src/Services/DownloadService
```

加载过程：

| 步骤 | 配置来源 | 说明 |
|------|---------|------|
| 1 | `appsettings.json` | 加载基础配置 |
| 2 | `appsettings.Development.json` | 默认 `ASPNETCORE_ENVIRONMENT=Development` |
| 3 | 系统环境变量 | 如果在系统中设置了对应变量则覆盖 |

此时 `appsettings.json` 中的 `CdnSettings` 直接生效（因为没有 Docker 环境变量覆盖）。

## 环境变量与 JSON 配置的映射关系

ASP.NET Core 使用 `__`（双下划线）分隔环境变量名来表示 JSON 的嵌套层级：

| 环境变量 | 等价 JSON 配置 |
|---------|---------------|
| `CdnSettings__BaseUrl` | `{ "CdnSettings": { "BaseUrl": "..." } }` |
| `CdnSettings__AuthKey` | `{ "CdnSettings": { "AuthKey": "..." } }` |
| `ConnectionStrings__DefaultConnection` | `{ "ConnectionStrings": { "DefaultConnection": "..." } }` |
| `JwtSettings__Secret` | `{ "JwtSettings": { "Secret": "..." } }` |

## 最佳实践

1. **`appsettings.json`**：存放非敏感的默认值和结构定义
2. **`appsettings.Development.json`**：存放本地开发专用配置（如本地数据库连接）
3. **Docker 环境变量**：在容器部署时覆盖关键配置
4. **`.env` 文件**：生产环境通过 `.env` 注入敏感信息（密钥、密码等），**不提交到代码仓库**

> **简单记忆**：环境变量 > `appsettings.{Env}.json` > `appsettings.json`
